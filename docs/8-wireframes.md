## 1. 디자인 원칙

- **미니멀리즘**: 불필요한 장식은 최소화하고 콘텐츠(할일)에 집중합니다.
- **직관성**: 사용자가 설명 없이도 기능을 유추하고 사용할 수 있어야 합니다.
- **빠른 인터랙션**: 최소한의 클릭과 입력으로 핵심 작업을 완료할 수 있도록 설계합니다.
- **시각적 계층 구조**: 중요도에 따라 정보의 우선순위를 명확하게 표현합니다. (예: 기한 임박)

---

## 2. 화면 목록

Phase 1 (3일 스프린트) 범위 내 필수 화면은 다음과 같습니다.

1.  **회원가입 화면** (`/signup`)
2.  **로그인 화면** (`/login`)
3.  **할일 목록 화면** (`/todos`)
4.  **404 페이지** (`*`)

---

## 3. 화면별 와이어프레임

### 3.1 회원가입 화면

**경로**: `/signup`
**목표**: 사용자가 서비스에 가입하도록 유도하며, 최소한의 정보만으로 가입 절차를 완료합니다.

```
┌──────────────────────────────────────────┐
│                                          │
│             Jan TodoList                 │
│                                          │
│  회원가입                                │
│  새로운 계정을 만들어 할일을 관리하세요.   │
│                                          │
│  이메일 주소                             │
│  [minjun.kim@example.com______________| │
│                                          │
│  비밀번호                                │
│  [**************************] (eye-icon)│
│  * 8자 이상 입력해주세요.                │
│                                          │
│  이름                                    │
│  [김민준________________________________] │
│                                          │
│  +-------------------------------------+ │
│  |              가입하기               | │
│  +-------------------------------------+ │
│                                          │
│       이미 계정이 있나요? 로그인 →      │
│                                          │
└──────────────────────────────────────────┘
```

**컴포넌트 설명**:
- **로고**: "Jan TodoList"
- **페이지 제목**: "회원가입" 및 보조 설명
- **입력 필드**: 이메일, 비밀번호, 이름
  - **이메일**: `type="email"`, 실시간 형식 검증
  - **비밀번호**: `type="password"`, 보기/숨기기 아이콘, 8자 이상 조건 표시
- **가입하기 버튼**: CTA(Call To Action) 버튼, 유효성 검사 통과 시 활성화
  - **상태**: 기본(Primary color), 비활성(Gray), 로딩(Spinner)
- **로그인 링크**: 로그인 페이지로 이동하는 텍스트 링크

│  [**************************] (eye-icon)│

**컴포넌트 설명**:

- **로그인 버튼**: CTA 버튼.
- **회원가입 링크**: 회원가입 페이지로 이동.

### 3.3 할일 목록 화면 (메인)

**경로**: `/todos`
**목표**: 사용자가 자신의 모든 할일을 한눈에 보고, 생성, 수정, 삭제, 완료 등 핵심 작업을 수행하는 메인 화면입니다.

#### 3.3.1 할일이 있을 경우 (Default View)

```
┌────────────────────────────────────────────────────────────────────────┐
│ Jan TodoList                                      김민준님 (로그아웃) │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  [ 오늘 할일은 무엇인가요?                     ] [ 기한 ] [  추가  ]    │
│  ────────────────────────────────────────────────────────────────────    │
│                                                                        │
│  □ 프로젝트 제출                  <오늘> (edit) (delete)               │
│                                                                        │
│  □ 교수님 미팅 준비               <오늘> (edit) (delete)               │
│                                                                        │
│  □ 논문 1차 초고 완성             <내일> (edit) (delete)               │
│                                                                        │
│  □ 병원 예약                      11/29 (금) (edit) (delete)           │
│                                                                        │
│  ✓ 장보기                         <어제> (edit) (delete)               │
│     (취소선, 회색 처리)                                                │
│                                                                        │
│  □ 책 읽기                                 (edit) (delete)           │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

**컴포넌트 설명**:
- **헤더**:
  - **로고**: 좌측 상단.
  - **사용자 정보**: 우측 상단, "OOO님" 및 "로그아웃" 버튼.
- **할일 추가 폼**:
  - **제목 입력**: `placeholder`="오늘 할일은 무엇인가요?"
  - **기한 선택**: Date Picker 컴포넌트.
  - **추가 버튼**: Enter 키로도 동작.
- **할일 목록**:
  - **정렬**: PRD에 정의된 기한 우선순위로 자동 정렬.
  - **항목**: `TodoItem` 컴포넌트의 리스트 (상세 설명은 4.1 참고).
    - **`<오늘>`**: 빨간색 텍스트.
    - **`<내일>`**: 주황색 텍스트.
    - **수정/삭제 아이콘**: 항목에 마우스 호버 시 표시.

## 4. 공통 컴포넌트

### 4.1 할일 항목 (TodoItem)

#### 상태 1: 기본 (미완료)
```
[ ]  프로젝트 제출                  <오늘> (edit) (delete)
```
- **체크박스**: 빈 사각형. 클릭 시 '완료' 상태로 변경.
- **제목**: 기본 텍스트 색상. 클릭 시 '편집' 상태로 변경.
- **기한**: 기한별로 다른 색상 적용.
- **아이콘**: 호버 시에만 표시.

#### 상태 2: 완료
```
[✓]  ~~프로젝트 제출~~              <오늘> (edit) (delete)
      (회색, 취소선)
```
- **체크박스**: 체크 아이콘. 클릭 시 '미완료' 상태로 변경.
- **제목**: 회색 + 취소선.

#### 상태 3: 편집
```
[ ]  [ 프로젝트 제출____________________ ] [ 11/26 ]  [ 저장 ] [ 취소 ]
```
- **제목**: 입력 필드로 변경.
- **기한**: Date Picker로 변경.
- **버튼**: 저장/취소 버튼 표시.
- **액션**: Enter 키로 저장, ESC 키로 취소.

---

### 4.2 삭제 확인 모달

**트리거**: 할일 항목의 `(delete)` 아이콘 클릭 시
**목표**: 사용자의 실수로 인한 데이터 삭제를 방지합니다.

```
            ┌───────────────────────────────┐
            │                               │
            │     정말 삭제하시겠습니까?      │
            │                               │
            │  "프로젝트 제출" 할일이        │
            │    영구적으로 삭제됩니다.     │
            │                               │
            │   [ 취소 ]   [    삭제    ]   │
            │                               │
            └───────────────────────────────┘
```
- **배경**: 모달 뒷 배경은 어둡게 처리 (Dimmed).
- **취소 버튼**: 기본 버튼.
- **삭제 버튼**: Danger color (빨간색).

## 5. 반응형 레이아웃

### 5.1 모바일 뷰 (`< 768px`)

**특징**:
- 전체적으로 세로 스크롤 기반의 1단 레이아웃.
- 할일 추가 기능은 하단 고정 버튼(FAB)으로 변경하여 접근성을 높임.

```
┌──────────────────┐
│ Jan Todo         │  ← 헤더: 로고만 or 간소화
│ (user-icon)      │  ← 사용자 메뉴 아이콘
├──────────────────┤
│ □ 프로젝트 제출  │
│   <오늘>         │
│                  │
│ □ 논문 초고 완성 │
│   <내일>         │
│                  │
│ ✓ 장보기         │
│   <어제>         │
│ ...              │
│ (스크롤 영역)    │
│                  │
│                  │
│                  │
│                  │
│                  │
│             (+)  │  ← 하단 고정 추가 버튼 (FAB)
└──────────────────┘
```

**모바일 인터랙션**:
- **할일 추가**: `(+)` 버튼 클릭 시 전체 화면을 덮는 입력 모달 표시.
- **수정/삭제**: 항목을 왼쪽으로 스와이프하면 수정/삭제 버튼 노출 (선택). 또는 항목 우측에 "더보기(...)" 아이콘 배치.
- **터치 영역**: 모든 버튼 및 인터랙션 요소는 최소 44x44px 크기 확보.
