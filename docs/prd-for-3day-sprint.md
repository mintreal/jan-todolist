**문서 버전**: 1.0
**작성일**: 2025-11-25
**대상 독자**: 개발팀, 기획자
**목표**: 3일 안에 개발할 핵심 기능 프로토타입의 요구사항을 명확히 정의

---

## 1. 프로젝트 개요

### 1.1 제품 목표
**Jan TodoList**는 사용자 인증 기반의 개인 할일 관리 웹 애플리케이션입니다. 이번 3일 스프린트의 목표는 사용자가 회원가입 후 자신의 할일을 생성하고, 기한에 따라 관리할 수 있는 **핵심 기능 프로토타입**을 완성하는 것입니다.

### 1.2 성공 기준
- 사용자는 이메일과 비밀번호로 회원가입 및 로그인을 할 수 있다.
- 로그인한 사용자는 자신의 할일을 생성, 조회, 수정, 삭제할 수 있다.
- 할일 목록은 기한이 가까운 순서대로 자동 정렬된다.
- 완성된 프로토타입은 웹을 통해 접근 가능하다.

## 2. MVP 기능 범위

`docs/4-3day-sprint-plan.md`에 따라 3일 프로토타입의 범위를 다음과 같이 정의합니다.

### 2.1 포함 기능 (In-Scope)

| 기능 | 사용자 스토리 | 우선순위 |
|---|---|---|
| **사용자 인증** | 사용자는 계정을 만들고 로그인하여 개인화된 할일 목록에 접근할 수 있다. | **P0** |
| **할일 생성 (Create)** | 사용자는 새로운 할일을 제목과 선택적 기한으로 추가할 수 있다. | **P0** |
| **할일 조회 (Read)** | 사용자는 자신의 모든 할일을 기한이 임박한 순서로 정렬된 목록으로 볼 수 있다. | **P0** |
| **할일 수정 (Update)** | 사용자는 기존 할일의 제목, 기한, 완료 여부를 변경할 수 있다. | **P0** |
| **할일 삭제 (Delete)** | 사용자는 더 이상 필요 없는 할일을 목록에서 제거할 수 있다. | **P0** |

### 2.2 제외 기능 (Out-of-Scope)

다음 기능들은 프로토타입의 복잡도를 낮추고 3일 내 완성을 위해 제외하며, Phase 2 이후에 구현합니다.

- **캘린더 뷰**: 월간 달력 형태의 뷰
- **알림 기능**: 기한 임박 시 브라우저 또는 이메일 알림
- **공유 기능**: 다른 사용자와 할일 공유
- **외부 캘린더 연동**: 구글, 네이버 캘린더 연동
- **고급 기능**: 할일 상세 설명, 우선순위 설정, 설정 페이지 등

## 3. 기능 명세

### 3.1 사용자 인증
- **회원가입**: `이메일`, `비밀번호`, `이름`을 입력받아 사용자를 생성합니다.
- **로그인**: `이메일`, `비밀번호`로 사용자를 확인하고, 성공 시 `JWT 토큰`을 발급합니다.
- **인증 유지**: 발급된 JWT 토큰을 사용하여 API 요청 권한을 확인합니다.

### 3.2 할일 관리 (CRUD)
- **데이터 모델**:
  - `TodoItem`: `id`, `user_id`, `title`, `due_date`, `is_completed`
- **생성 (Create)**:
  - `title`(필수), `due_date`(선택)을 입력받아 새로운 할일을 생성합니다. `is_completed`의 기본값은 `false`입니다.
- **조회 (Read)**:
  - 사용자의 모든 할일을 단일 목록으로 반환합니다.
  - **정렬 순서**: 1. `due_date` 오름차순 (가까운 날짜 먼저) 2. `due_date`가 없는 경우 맨 뒤로
- **수정 (Update)**:
  - `title`, `due_date`, `is_completed` 상태를 변경할 수 있습니다.
- **삭제 (Delete)**:
  - 지정된 `id`의 할일을 데이터베이스에서 삭제합니다.

## 4. 비기능 요구사항

### 4.1 기술 스택 (단순화 버전)
- **백엔드**: JavaScript, Node.js, Express.js
- **프론트엔드**: JavaScript, React (Vite), `useState` 기반 상태관리, Tailwind CSS
- **데이터베이스**: SQLite (로컬 개발) / PostgreSQL (배포 시)
- **인증**: `jsonwebtoken`, `bcrypt` 라이브러리 활용

### 4.2 UI/UX
- **디자인**: Tailwind CSS를 활용한 최소한의 깔끔한 디자인.
- **레이아웃**: 단일 페이지 애플리케이션(SPA)으로, 로그인/회원가입 화면과 메인 할일 목록 화면으로 구성됩니다.
- **반응형**: 데스크톱 웹 브라우저에서 사용 가능한 수준을 목표로 합니다.

## 5. 데이터베이스 및 API

### 5.1 데이터베이스 스키마
`docs/4-3day-sprint-plan.md`의 스키마를 따릅니다.

- **users**: `id`, `email`, `password`, `name`, `created_at`
- **todos**: `id`, `user_id`, `title`, `due_date`, `is_completed`, `created_at`

### 5.2 API 엔드포인트 (핵심)
- `POST /api/auth/register`: 회원가입
- `POST /api/auth/login`: 로그인
- `GET /api/todos`: 할일 목록 조회
- `POST /api/todos`: 할일 생성
- `PATCH /api/todos/:id`: 할일 수정
- `DELETE /api/todos/:id`: 할일 삭제
